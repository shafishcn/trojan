# 控制中心恢复演练清单

这份清单用于定期验证控制中心的备份、恢复和上线切换流程是否真的可用。

建议频率：

- 最少每季度一次
- 大版本升级前额外执行一次
- 关键权限或数据模型变更后补做一次

## 演练目标

- 验证最近备份可成功导出
- 验证备份可在独立测试环境导入
- 验证管理员、节点、用户、绑定、任务和审计可以恢复
- 验证控制中心在恢复后仍能通过 `/readyz`、`/metrics` 和告警摘要接口工作

## 演练前准备

- 准备一台独立测试环境
- 准备测试 MySQL
- 准备最近一次控制中心备份
- 准备一个可用的管理员 token

## 演练步骤

1. 导出一份最新备份
   - 调用 `GET /api/control/backup/export`
   - 记录备份文件名和导出时间

2. 启动测试控制中心
   - 用独立 MySQL 启动 `trojan control`
   - 确认 migration 正常执行

3. 导入备份
   - 调用 `POST /api/control/backup/import`
   - 记录返回结果

4. 验证核心数据
   - 管理员数量是否正确
   - 节点数量是否正确
   - 用户数量是否正确
   - 用户绑定是否正确
   - 历史任务和审计是否存在

5. 验证运行接口
   - `GET /readyz`
   - `GET /metrics`
   - `GET /api/control/runtime/status`
   - `GET /api/control/alerts/summary`

6. 验证管理功能
   - 管理员登录
   - 新建一个测试用户
   - 给测试节点下发一个 `sync_users` 任务
   - 验证任务状态流转

7. 记录结果
   - 演练时间
   - 参与人
   - 备份文件名
   - 是否成功
   - 遇到的问题
   - 下次改进项

## 验收标准

- 恢复环境能正常登录控制中心
- 核心数据与备份内容一致
- 健康检查和指标接口正常
- 可以成功创建并执行新任务
