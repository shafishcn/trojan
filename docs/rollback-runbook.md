# 控制中心回滚手册

这份手册用于控制中心或 Agent 升级后出现异常时的快速回滚。

## 触发条件

符合以下任一条件即可进入回滚判断：

- `/readyz` 持续失败
- 控制台无法登录
- 大量节点无法心跳
- 任务大面积失败
- migration 后核心数据异常
- 指标和告警出现明显异常放大

## 回滚前判断

先确认问题范围：

- 只有控制中心异常
- 只有少量 Agent 异常
- 控制中心和 Agent 同时异常
- 数据结构或 migration 异常

如果只是少量 Agent 异常，不要急着整体回滚控制中心。

## 快速回滚路径

### 场景一：控制中心二进制异常

1. 停止当前控制中心
2. 恢复上一版本二进制
3. 启动控制中心
4. 检查：
   - `systemctl status`
   - `/readyz`
   - 管理员登录
   - `/api/control/runtime/status`

### 场景二：控制中心数据异常

1. 停止当前控制中心
2. 准备最近一次控制中心备份
3. 恢复 MySQL 或启动独立恢复环境
4. 通过 `POST /api/control/backup/import` 导入备份
5. 启动控制中心
6. 验证管理员、节点、用户、绑定、任务和审计

### 场景三：部分 Agent 升级异常

1. 停止继续升级剩余 Agent
2. 把异常节点回滚到上一版本 Agent
3. 验证：
   - 节点注册
   - 心跳
   - 拉任务
   - 结果回报

## 回滚后的验证

- `/readyz` 返回正常
- `/metrics` 可以被正常抓取
- `/api/control/alerts/summary` 没有持续恶化
- 至少 1 个节点能成功执行测试任务
- 管理台可正常查看节点、用户、任务和审计

## 何时停止回滚

满足以下条件可结束回滚：

- 控制中心稳定运行 30 分钟以上
- 节点在线率恢复
- 新失败任务没有继续增加
- 管理员可以正常完成核心操作

## 回滚后复盘

建议记录：

- 异常开始时间
- 影响范围
- 回滚开始和结束时间
- 使用的回滚版本
- 使用的备份文件
- 根因分析
- 后续修复计划
